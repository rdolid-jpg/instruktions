🎯 Налаштування Stratifications для A/B тесту

📁 1. Створення кампанії та налаштування тригерів

📌 Крок 1.1: Створіть нову кампанію

У розділі Realtime Campaigns/Scheduled Campaigns → Create Campaign → оберіть тип кампанії.

📌 Крок 1.2: Налаштуйте тригер

Додайте подію, яка буде запускати кампанію. Наприклад:

  • User Login — вхід користувача
  • Custom Event — будь-яка кастомна подія
  • Segment Entry — вхід у сегмент

📌 Крок 1.3: Створіть Flow з Update Property

У конструкторі флоу додайте блок Update Property:

  • В полі Property виберіть "Core: JS markers"
  • В полі Value вставте назву відповідного Label tags
Name Label tags Що робить Куди вставляти
{{label.func_strat_mark_user_first_level}} Створює базовий тег (сегмент + депозит) Перший Update Property у флоу
{{func_strat_mark_user_second_level}} Створює універсальний тег (title) Другий Update Property у флоу
{{func_strat_mark_user_third_level}} Створює груповий тег (test/control) Третій Update Property у флоу

Або просто склонуйте вже налаштовану кампанію для прикладу example_setting_campaign_for_stratification і відредагуйте необхібні налаштування тригеру

💡 Важливо:

Блоки Update Property мають йти послідовно один за одним. Спочатку перший рівень, потім другий, потім третій.


⚙️ 2. Налаштування параметрів функцій (при необхідності)

📌 Як змінити параметри в функціях в Label tags

У кожній функції є блок НАЛАШТУВАННЯ на початку. Ось що можна змінити для нової/поточної кампанії:

🔧 Змінні, які можна налаштовувати:

Змінна Де знаходиться Що робить Приклад
VAR_PROJECT Всі 3 функції Назва кампанії (унікальний ідентифікатор) 'previp_daily_bonuses'
segmentCodes Функція 1 Відповідність кодів сегментів назвам '1559': 'Iron'
avgDepRanges Функція 1 Діапазони середнього депозиту { min: 0, max: 30, suffix: 'avg_dep_30' }
exchangeRates Функція 1 Курси валют для конвертації в USD EUR: 1.1

📝 Як додати новий сегмент:

var segmentCodes = {
    '205': 'Petrovich',  // Новий сегмент
    '228': 'Ace',        // Існуючий
    '1559': 'Iron'       // Існуючий (найвищий пріоритет)
};

💡 Важливо: Чим нижче в списку — тим ВИЩИЙ пріоритет! Останній сегмент у списку має найвищий пріоритет.

📝 Як змінити діапазони депозитів:

var avgDepRanges = [
    { min: 0, max: 29.999999, suffix: 'avg_dep_30' },      // 0-30
    { min: 30, max: 50, suffix: 'avg_dep_30_50' },         // 30-50
    { min: 50.000001, max: Infinity, suffix: 'avg_dep_51+' } // 51+
];

📝 Як змінити курси валют:

var exchangeRates = {
    USD: 1,
    EUR: 1.1,    // 1 EUR = 1.1 USD
    GBP: 1.20,   // 1 GBP = 1.20 USD
    CAD: 0.7     // 1 CAD = 0.7 USD
};

⚠️ Увага!

Якщо ви змінюєте назву VAR_PROJECT — потрібно змінити його ОДИНАКОВО у всіх трьох функціях! Інакше теги не будуть пов'язані між собою.

🗑️ 3. Видалення тегів (якщо потрібно прибрати користувача з тесту, або почистити теги по закінченню тесту)

📌 Крок 3.1: Створіть окрему кампанію для видалення

Створіть нову Campaign з тригером, який спрацьовує для потрібних користувачів.

📌 Крок 3.2: Додайте блок Remove JS markers

У флоу додайте блок Remove JS markers:

📌 Крок 3.3: Вкажіть теги для видалення

В полі Remove marks from user додайте назви тегів які потрібно видалити:

ab_test_previp_daily_bonuses_strata_iron_avg_dep_30,
ab_test_previp_daily_bonuses_title,
ab_test_previp_daily_bonuses_group_test

💡 Можна вказати кілька тегів через кому.

⚠️ Увага!

Після видалення тегів користувач знову зможе потрапити в тест при наступному запуску кампанії.

📄 4. Що можна змінювати в коді функції, не боячись зломати логіку обчислення

🔵 Функція 1: Створення тегу першого рівня

(function() {
    // ========== НАЛАШТУВАННЯ ==========
    var VAR_PROJECT = 'previp_daily_bonuses';  // <--- можна змінити назву в одинарних лапках '' 
    var STATIC_PREFIX = 'ab_test';  // <--- можна змінити назву в одинарних лапках '' 
    var STATIC_MIDDLE = 'strata';  // <--- можна змінити назву в одинарних лапках '' 
    
    var exchangeRates = {
        USD: 1, CAD: 0.7, AUD: 0.7, EUR: 1.1, GBP: 1.20, NOK: 0.1  // <--- можна змінити значення курсу, також додати нову валюту, не важливо в якому місці
    };
    
    var segmentCodes = {
        '205': 'Petrovich',  // <--- можна змінити назву, або код в одинарних лапках '', 
        '228': 'Ace',  // також додати новий екстернал сегмент, або прибрати існуючий, 
        '762': 'Ace2',  // тут коди екстернал сегментів повинні співпадати з кодами в сегменті який буде використаний в кампані, 
        '750': 'Ace3',  // а також потрібно зберігати пріоритетність сегментів, 
        '256': 'Star',  // чим нижчий сегмент тим він має більший пріоритет над іншими.
        '1559': 'Iron'  
    };
    
    var avgDepRanges = [
        { min: 0, max: 29.999999, suffix: 'avg_dep_30' },  // <--- можна змінити значення мін макс, 
        { min: 30, max: 50, suffix: 'avg_dep_30_50' },  // також назву суфіксу в одинаїрних лапках ''
        { min: 50.000001, max: Infinity, suffix: 'avg_dep_51+' }
    ];
    // ===================================
    ...
   })();
