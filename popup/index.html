🎯 Pop-up у Smartico

Від готового шаблону до кастомного рішення: повний гайд

📦 1. Базовий pop-up: копіюємо та налаштовуємо

Що потрібно зробити

  • Перейдіть у Marketing → Templates → Popups.
  • Знайдіть готовий шаблон (наприклад, “Example Popup with HTML/CSS/JS”) та натисніть Clone.
  • Відредагуйте поля в Configuration (JSON): змініть текст, посилання на картинку, URL кнопки тощо.
  • У вкладці Body за потреби підкорегуйте стилі (CSS).

💡 Основні параметри, які можна міняти одразу

title_text, subtitle_text, button_text, button_url, hero_image_url, background_color, title_color тощо.

// Приклад частини JSON-конфігурації
{
    "name": "title_text",
    "type": "text",
    "label": "Title",
    "value": "Ваш новий заголовок"
}

✏️ 2. Як додати або видалити поле у шаблоні

Структура JSON

Кожне поле описується об'єктом у масиві fields. Ви можете додати нове поле або видалити непотрібне прямо у редакторі Configuration.

📌 Додаємо новий текстовий параметр

{
    "name": "new_field",
    "type": "text",
    "label": "Мій новий напис",
    "value": "Текст за замовчуванням"
}

Після додавання в JSON, у HTML-коді (Body) використайте змінну {%#o.new_field%} для виводу її значення.

📌 Видаляємо поле

Просто приберіть відповідний об'єкт з масиву fields у JSON, а з HTML видаліть рядок, де ця змінна використовується.

⚠️ Важливо

Не видаляйте поля, які використовуються у HTML-розмітці, не прибравши їх звідти — це може зламати відображення.

🌍 3. Content Variations: мови, гео та кастомні умови

У pop-up ассеті є вкладка Content Variations. Вона дозволяє створювати декілька версій контенту для одного попапу, які показуватимуться залежно від мови, країни чи будь-якої іншої умови.

Що можна змінювати у варіаціях?

  • Будь-який текст (заголовок, підпис, текст кнопки).
  • Посилання на зображення (hero image).
  • Кольори (фон, текст, кнопка).
  • Посилання для CTA.

Як створити варіації

  1. Відкрийте pop-up ассет, перейдіть у вкладку Content Variations.
  2. Натисніть Add Variation.
  3. Вкажіть умову (наприклад, Language = fr або Country = CA).
  4. У формі, що з'явиться, змініть потрібні поля (текст, URL картинки, кольори).
  5. Збережіть. Тепер для користувачів, що підпадають під умову, покажеться цей варіант.

💡 Приклад: два варіанти — англійська та французька

Створіть варіацію French із умовою Language = fr. Змініть title_text на французький переклад, решту полів за потреби. Базова версія залишиться для всіх інших мов.

// JSON варіації (спрощено)
{
    "title_text": "Votre bonus",
    "button_text": "Obtenir",
    "hero_image_url": "https://cdn.example.com/bonus_fr.png"
}

🛠️ 4. Створення кастомного попапу з власним HTML/CSS

Щоб повністю контролювати дизайн, використовуйте клонування вже готовіх шаблонів які є в Smartico. або скористайтесть допомогою AI, і виберіть кнопку "Create With AI" і в промпті детально опишіть свій запит по попапу з описом структури кольорів і функціоналу папапу. І у вкладці Body ви можете відредагувати неточності або доповнити верстку попапа і замінити картинки і посилання.

Основні правила

  • Контент попапу має бути всередині <div id="popup">...</div> — Smartico автоматично вставляє цю обгортку.
  • Динамічні значення з JSON вставляються так: {%#o.назва_поля%}.
  • Стилі пишіть у тегу <style> всередині Body.

✅ Рекомендована структура Body

<style>
  /* ваші CSS-правила */
</style>

<!-- HTML-розмітка попапу -->
<div class="my-popup">
  <img src="{%#o.hero_image_url%}">
  <h1>{%#o.title_text%}</h1>
  <a href="{%#o.button_url%}">{%#o.button_text%}</a>
</div>

<script>
  {{label.script_for_popup_under_modal}}
</script>

🚨 5. УВАГА! Попап не має перекривати модальні вікна на сайті

Щоб pop-up Smartico не відображався поверх відкритих модальних вікон на сайті або в профілі користувача (наприклад, вікно депозиту), у кожен шаблон попапу необхідно додати спеціальний скрипт в самий кінець поля Body.

⚠️ Обов'язково вставити цей код

Навіть якщо у вашому Body вже є теги <script>, додайте цей блок окремо в самому низу.

<script>
{{label.script_for_popup_under_modal}}
</script>

Цей Label Tag містить готовий JS-код, який перевіряє наявність модального вікна та за потреби знижує z-index фрейму, щоб попап опинився під ним.

📌 Важливо

  • Код обов'язково має бути вставлений після всього іншого HTML/CSS у полі Body.
  • Не змінюйте форматування — використовуйте точнісінько як у прикладі.
  • Якщо ви створюєте кілька кастомних шаблонів, додавайте цей скрипт у кожен.
